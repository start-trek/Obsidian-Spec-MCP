# Templater Spec Pack

- Templater Commands: https://silentvoid13.github.io/Templater/commands/overview.html

- User Functions: https://silentvoid13.github.io/Templater/user-functions/overview.html

Purpose
- Generate Templater-aware note templates with proper command syntax, balanced tags, and correct use of internal and user-defined functions.

## Command Types

Interpolation (`<% ... %>`)
- Evaluates the expression and inserts the result into the note.
- Use for reading values: `<% tp.file.title %>`, `<% tp.date.now("YYYY-MM-DD") %>`.

Execution (`<%* ... %>`)
- Runs JavaScript without inserting a return value.
- Use for variable assignments, control flow, and side effects.
- Example: `<%* const today = tp.date.now("YYYY-MM-DD") %>`

Whitespace control (`<%- ... %>`)
- Trims leading whitespace from the output line.

Rules
- Opening and closing tags **must** be balanced. Every `<%` needs a matching `%>`.
- Do not nest Templater tags.
- Prefer `tp.*` internal functions over raw JavaScript when available.

## Internal Functions

tp.file
- `tp.file.title` — Current note title.
- `tp.file.path()` — Full vault path.
- `tp.file.folder()` — Parent folder name.
- `tp.file.creation_date(format)` — File creation date.
- `tp.file.cursor()` — Place cursor here after template insertion.
- `tp.file.rename(new_name)` — Rename the file.

tp.date
- `tp.date.now(format, offset?, reference?, reference_format?)` — Current date/time.
- `tp.date.tomorrow(format)` — Tomorrow's date.
- `tp.date.yesterday(format)` — Yesterday's date.
- Common format: `"YYYY-MM-DD"`, `"YYYY-MM-DD HH:mm"`.

tp.system
- `tp.system.prompt(prompt_text)` — Show an input dialog.
- `tp.system.suggester(labels, values)` — Show a selection dialog.
- `tp.system.clipboard()` — Read clipboard contents.

## User Functions

Rules
- User scripts are called via `tp.user.scriptName(args)`.
- Scripts must be registered in Templater settings (or runtime config `templater_user_scripts`).
- Referencing an unregistered script triggers a validation warning.

Examples
- `<% tp.user.formatDate("2026-04-01") %>`
- `<%* const result = await tp.user.slugify(tp.file.title) %>`

## Folder Templates

Rules
- Templater can auto-apply templates when creating notes in specific folders.
- Configured via `templater_folder_templates` in the profile.
- When generating a template for a folder with an assigned template, match that template's structure.

## Full Template Example

```
---
created: <% tp.file.creation_date("YYYY-MM-DD") %>
title: <% tp.file.title %>
tags:
  - draft
---

# <% tp.file.title %>

<%* const today = tp.date.now("YYYY-MM-DD") %>

## Summary
Created on: <% today %>

<% tp.file.cursor() %>
```

Edge cases
- Unbalanced tags (missing `%>`) cause template rendering to fail.
- Avoid `<% tp.file.title` without closing `%>`.
- Execution tags with syntax errors will halt template processing.

## Cross-pack notes
- Templater output should survive Linter validation (single H1, clean spacing).
- Tasks syntax can be embedded in templates: `- [ ] <% tp.system.prompt("Task?") %> 📅 <% tp.date.now("YYYY-MM-DD") %>`
