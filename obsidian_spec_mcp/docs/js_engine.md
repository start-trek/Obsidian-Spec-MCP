# JS Engine Spec Pack

Sources
- JS Engine Docs: https://www.moritzjung.dev/obsidian-js-engine-plugin-docs/
- JS Engine GitHub: https://github.com/mProjectsCode/obsidian-js-engine-plugin
- API Reference: https://www.moritzjung.dev/obsidian-js-engine-plugin-docs/api/

Purpose
- Describe executable `js-engine` fenced code blocks that run JavaScript inside Obsidian notes, producing dynamic rendered output.

## Code Blocks

Rules
- JS Engine code **must** be inside fenced code blocks with language `js-engine`.
- The block should `return` a renderable value (typically via `engine.markdown.create()`).
- Every opening ` ```js-engine ` must have a matching closing ` ``` `. Unclosed blocks are errors.
- Code runs in a sandboxed context with access to the `engine` API and the Obsidian app object.

## Engine API

engine.markdown
- `engine.markdown.create(markdownString)` — Render a markdown string.
- `engine.markdown.createBuilder()` — Create a markdown builder for complex output.

engine.internal
- `engine.internal.getObsidianModule(name)` — Access an Obsidian API module.
- `engine.internal.executeFile(filePath)` — Execute another JS Engine script file.

context
- `context.file` — The current file object.
- `context.metadata` — The current note's metadata/frontmatter.
- `app` — The Obsidian `App` object (full API access).

## Helpers

Rules
- User-defined helper scripts can be registered via `js_engine_helpers` in the profile.
- When helpers are configured, generated snippets include a reference to the first available helper.
- Referencing a non-existent helper won't cause a validation error but may fail at runtime.

## Examples

Basic block
````
```js-engine
return engine.markdown.create('*Hello from JS Engine*');
```
````

Multi-line with date
````
```js-engine
const today = new Date().toISOString().slice(0, 10);
const greeting = `Today is ${today}`;
return engine.markdown.create(greeting);
```
````

Using a builder
````
```js-engine
const mb = engine.markdown.createBuilder();
mb.createHeading(2, 'Dynamic Section');
mb.createParagraph('This was generated by JS Engine.');
return mb;
```
````

Accessing metadata
````
```js-engine
const title = context.metadata.frontmatter?.title || context.file.name;
return engine.markdown.create(`# ${title}`);
```
````

Edge cases
- Unclosed blocks (missing closing ```) are flagged as errors.
- Syntax errors inside the block will fail at runtime, not during validation.
- Very long scripts should be moved to external `.js` files and executed via `engine.internal.executeFile()`.

## Cross-pack notes
- JS Engine blocks are not affected by Templater processing (they run independently).
- Linter will not modify content inside fenced code blocks.
- JS Engine can dynamically generate Tasks syntax, but the output won't be validated by the Tasks plugin until rendered.
